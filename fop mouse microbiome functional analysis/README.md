# FOP Mouse Microbiome Functional Analysis (HUMAnN)

This repository contains code and processed input data required to reproduce functional metagenomic analyses associated with the submitted manuscript examining the gut microbiome in a mouse model of Fibrodysplasia Ossificans Progressiva (FOP).

## Data Generation

Raw sequencing data were processed using a shotgun metagenomic pipeline executed on UCSF's Wynton high-performance computing cluster. The full pipeline is available at: [https://github.com/ethan-dinh/HPC-Metagenomics/tree/main](https://github.com/ethan-dinh/HPC-Metagenomics/tree/main)

The pipeline performs the following steps in order:
1. **Quality trimming** — adapter and low-quality base removal with Trimmomatic
2. **Host decontamination** — removal of mouse (*C57BL/6NJ*) reads using KneadData
3. **Functional profiling** — gene family and pathway abundance estimation with HUMAnN3 against the UniRef90 protein database
4. **KO annotation** — KEGG Orthology (KO) mapping via HUMAnN3's regrouping utility

## Repository Structure

## Data Files

### Metadata — `data/Hsiao_shotgun_mapping_v3_final.csv`
The primary sample metadata table used throughout the analysis. Each row corresponds to one sequenced sample and includes:
- **Sample identifiers and group assignments** — treatment group (`CTRL`, `FOP`, `CTRL_ABX`, `FOP_ABX`), timepoint (`T1` = Baseline, `T2` = Endpoint), and antibiotic treatment status
- **Mouse-level covariates** — genotype (`PDGFRA-cre`, `ACVR1R206H`, `PDGFRA-cre x ACVR1R206H`, `WT`), sex, mouse ID, date of birth, and date of collection
- **Sequencing QC metrics** — raw read counts, clean read counts after KneadData processing, and GC percentage

This file is used to annotate all downstream differential abundance and pathway analyses.

### HUMAnN KO Abundance Table — `data/Humann/all_samples_ko_cpm.tsv`
A merged TSV table produced by HUMAnN3 containing KEGG Orthology (KO) gene family abundances in copies per million (CPM) across all samples. Rows represent KO terms (e.g., `K00003`) or stratified entries (`KO|g__Genus.s__species`); columns represent individual samples. This is the primary input for differential abundance analysis.

### KEGG KO Classification Table — `data/Overall-KEGG_KO_classification_table_Hsiao_FOP_metagenomics.csv`
A reference annotation table mapping each detected KO term to its KEGG hierarchical classification. Each row contains:

| Column | Description |
|--------|-------------|
| `KO` | KEGG Orthology identifier (e.g., `K00003`) |
| `KO_Class` | Top-level KEGG category (e.g., `Metabolism`) |
| `KO_Subclass` | Second-level category (e.g., `Carbohydrate metabolism`) |
| `KO_Subclass_2` | Third-level pathway (e.g., `Pentose and glucuronate interconversions`) |
| `KO_Name` | Gene name and function description |

This table is used to annotate and group KO results by biological pathway for heatmaps and enrichment figures.

## Required Software

- R (4.2+ recommended)
- RStudio
- Key R packages: `DESeq2`, `ggplot2`, `ggpubr`, `ComplexHeatmap`, `dplyr`, `tidyr`

All required R packages are installed automatically in the setup section of the notebook.

## Notes

All file paths are relative and should run when the folder structure is preserved. Set the working directory to the `analysis/` folder before knitting.
