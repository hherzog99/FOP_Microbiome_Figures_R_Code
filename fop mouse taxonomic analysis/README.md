# FOP Mouse Microbiome Taxonomic Analysis

This repository contains code and processed input data required to reproduce taxonomic microbiome analyses associated with the submitted manuscript examining the gut microbiome in a mouse model of Fibrodysplasia Ossificans Progressiva (FOP).

## Data Generation

Raw sequencing data were processed using a shotgun metagenomic pipeline executed on UCSF's Wynton high-performance computing cluster. The full pipeline is available at: [https://github.com/ethan-dinh/HPC-Metagenomics/tree/main](https://github.com/ethan-dinh/HPC-Metagenomics/tree/main)

The pipeline performs the following steps in order:
1. **Quality trimming** — adapter and low-quality base removal with Trimmomatic
2. **Host decontamination** — removal of mouse (*C57BL/6NJ*) reads using KneadData
3. **Taxonomic classification** — read-level classification with Kraken2 against a reference database
4. **Abundance re-estimation** — Bracken re-estimates species-level abundances from Kraken2 reports

## Data Files

### Metadata — `data/Hsiao_shotgun_mapping_v3_final.csv`
The primary sample metadata table used throughout the analysis. Each row corresponds to one sequenced sample and includes:
- **Sample identifiers and group assignments** — treatment group (`CTRL`, `FOP`, `CTRL_ABX`, `FOP_ABX`), timepoint (`T1` = Baseline, `T2` = Endpoint), and antibiotic treatment status
- **Mouse-level covariates** — genotype (`PDGFRA-cre`, `ACVR1R206H`, `PDGFRA-cre x ACVR1R206H`, `WT`), sex, mouse ID, date of birth, and date of collection
- **Sequencing QC metrics** — raw read counts, clean read counts after KneadData processing, and GC percentage

This file is used to annotate all downstream diversity and differential abundance analyses.

### Bracken Abundance Tables — `data/BrackenComplete/`
One TSV file per sample per taxonomic level, produced by Bracken after Kraken2 classification. Each file contains estimated read counts and fractional abundances for taxa at that level. Files follow the naming convention `<sample_id>_bracken_<level>.tsv`, where `<level>` is one of:

| Code | Taxonomic Level |
|------|----------------|
| `D`  | Domain |
| `K`  | Kingdom |
| `P`  | Phylum |
| `C`  | Class |
| `O`  | Order |
| `F`  | Family |
| `G`  | Genus |
| `S`  | Species |

The analysis primarily uses species-level (`_S`) files to build the OTU table for phyloseq. Bracken improves on raw Kraken2 counts by redistributing reads assigned to higher-level nodes down to the most probable species, yielding more accurate species-level abundance estimates.

### KneadData QC Reports — `data/Kneaddata/`
One TSV file per sample, produced by KneadData during host decontamination. Each file records paired-read counts at each stage of processing: raw input, after Trimmomatic quality trimming, after alignment-based removal of mouse (*C57BL/6NJ*) reads, and final usable read counts. These files are used to assess the proportion of host vs. microbial reads and overall sequencing depth per sample.

### Taxonomy Cache — `data/taxonomy_cache.rds`
An R object caching NCBI taxonomic lineage lookups (retrieved via the `taxize` package) for all species detected in the dataset. This avoids repeated API calls to NCBI during analysis and ensures reproducibility across sessions.

## Notes

All file paths are relative and should run when the folder structure is preserved. Set the working directory to the `analysis/` folder before knitting.
